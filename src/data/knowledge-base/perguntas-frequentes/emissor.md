# Emissor

O emissor será acessado em [emissor.yooga.com.br](http://emissor.yooga.com.br/)

## Vídeo de treinamento:

- Certificado Digital A1;
- Empresa no Simples Nacional - [Receita Federal](https://www8.receita.fazenda.gov.br/simplesnacional/aplicacoes.aspx?id=21);
- CSC (Código de Segurança do Contribuinte );
- Última numeração e série da NFe e NFCe - **Clientes já emitentes de notas
- Inscrição estadual ativa;
- Marca e modelo de MFE- **Apenas CE;
- Planilha de códigos fiscais enviados para Yooga.


A ativação do módulo fiscal no emissor pode ser realizada pelo usuário temporário, sem necessidade do acesso remoto. A aba é habilitada ao clicar 3x sobre ‘’fiscal’’ → configurações fiscais.

Os dados para o cadastro do cliente podem ser obtidos diretamente no pipe e no cadastro dele no sintegra/cadastro centralizado de contribuintes: [https://dfe-portal.svrs.rs.gov.br/NFE/CCC](https://dfe-portal.svrs.rs.gov.br/NFE/CCC) ou [https://consopt.www8.receita.fazenda.gov.br/consultaoptantes](https://consopt.www8.receita.fazenda.gov.br/consultaoptantes)

Os dados devem ser preenchidos de acordo com o cadastro do contribuinte, e a principio as opções de Simples Nacional e ID token não devem ser modificadas. É importante que o numero da nota cadastrada no sistema seja sempre o numero da ultima nota +1 para que não exista duplicidade, ou seja, se a ultima nota do cliente informada for a nº2000, no sistema vamos cadastrar como sendo 2001, da mesma forma caso o cliente não tenha emitido notas ainda, a numeração cadastrada será a 1.

A primeira serve para que o robô comece a importar as notas de compra do cliente para o sistema → [e](http://painel.yooga.com.br/)missor.yooga.com.br → fiscal → notas de compra.

A segunda opção manifestar as notas automaticamente,  serve para que o robô informe a Sefaz a ciência da emissão da nota e assim capturamos o XML das notas de compras.


Feito Salve as configurações para concluir a ativação.

> [!NOTE]
> **Importante
É obrigatório realizar pelo menos duas vendas de teste no sistema do cliente e garantir que a nota fiscal foi autorizada, logo em seguida realizar o cancelamento das mesmas.

É necessário configurar a impressão dos cupons quando o cliente é emissor.


Após feito as vendas informar para o cliente que foi ativado com sucesso ou informar a falha.

Em caso de falha, consulte : [Principais erros na emissão de notas](https://www.notion.so/24300ee50e90810a8f36cd886c2a5d2c)
